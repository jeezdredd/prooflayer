# Vision LLM Analyzer

Multimodal LLM examines image and returns structured verdict + reasoning. Probes lighting consistency, texture coherence, fine details (fingers/eyes/text), backgrounds, uncanny valley.

## MIME

`image/jpeg`, `image/png`, `image/webp`

## Model selection

Reads `settings.OLLAMA_VISION_MODEL` (env-driven). Current dev: `qwen2.5vl:3b`.

| Model | Size | Notes |
|-------|------|-------|
| `qwen2.5vl:3b` | 3.2 GB | Default. Fits 12GB Docker comfortably. |
| `qwen2.5vl:7b` | 6.0 GB | Better quality but OOMs alongside text LLM on 12GB. Pulled but not active. |
| `moondream` | 1.7 GB | Original default. Lower fidelity. Still pulled. |
| `llava:7b` | 4.7 GB | Fallback in code default if env unset. |

Config: see [[infrastructure/env-vars]].

## Encoding

```python
MAX_VISION_DIM = 1024
VISION_JPEG_QUALITY = 88

img = Image.open(file_path).convert("RGB")
img.thumbnail((1024, 1024))  # preserve aspect
buf = io.BytesIO()
img.save(buf, format="JPEG", quality=88)
image_b64 = base64.b64encode(buf.getvalue()).decode()
```

## Prompt (LLM_IMAGE_PROMPT)

Asks the model to inspect 5 dimensions:
1. Lighting consistency
2. Texture coherence
3. Fine details (fingers, teeth, eyes, ears, text)
4. Background plausibility
5. Overall aesthetic / uncanny

Response forced to JSON via Ollama `format: "json"`:

```json
{"verdict": "ai_generated|human_photo|uncertain", "confidence": 0.0-1.0, "reasoning": "2-3 full sentences..."}
```

## HTTP call

`POST {OLLAMA_URL}/api/generate` with `timeout=300`. `stream=False`. `images: [b64]`.

## Response parsing

`_parse_vision_response(raw)`:

1. Try JSON regex `\{.*\}` -> parse -> validate `verdict in VERDICT_CONFIDENCE` and reasoning length >= 20 chars
2. Fallback: lowercase substring match
   - `ai_signals`: "ai-generated", "ai generated", "synthetic image", "artificially generated", "generated by ai", "deepfake", "this is fake", "looks fake", "computer-generated"
   - `human_signals`: "real photo", "authentic photo", "genuine photo", "human photo", "not ai-generated", "natural photograph", "real photograph", "looks authentic", "looks real"
3. If neither matches with sufficient text -> `model_failure`

Confidence map:

```python
VERDICT_CONFIDENCE = {
    "ai_generated": 0.6,
    "human_photo":  0.6,
    "uncertain":    0.5,
}
```

## Verdict mapping (LLM verdict -> analysis verdict)

```
ai_generated  -> fake
human_photo   -> authentic
model_failure -> error
uncertain     -> inconclusive
```

## Evidence shape

```json
{
  "llm_verdict": "ai_generated" | "human_photo" | "uncertain" | "model_failure",
  "reasoning": "2-3 sentences..."
}
```

> [!key-insight] Model name stripped
> Evidence used to include `"model": "moondream"`. Removed because user does not want model names exposed in UI. See [[fixes/model-name-leak]].

## Cold-load latency

First inference after `OLLAMA_KEEP_ALIVE` timeout (`2m`) cold-loads weights into RAM. ~10-15s for 3b, ~60s+ for 7b on CPU. See [[concepts/memory-budget]].
